# 常见问题

## 账户与登录

### 我可以不授权邮箱访问直接登录吗？

不可以。Mp2RSS 以邮箱作为账户唯一标识，OAuth 回调必须返回可用邮箱，否则登录会被拒绝。请先在 OAuth 提供方处把邮箱设为公开或完成邮箱验证后再重试。

### 我先后用 GitHub 和 Google 登录，会被建成两个账户吗？

不会。只要两次登录返回的是同一个邮箱，就会落到**同一个账户**，订阅和会员权益互通。

## 订阅相关

### 单个账户最多能订阅多少公众号？

**400 个**。

### 取消订阅后再订阅会浪费配额吗？

重新订阅同一公众号，不占用新的订阅配额；

### 我能从某篇文章直接反查并订阅吗？

可以。在"添加订阅"里贴入任意一篇公众号文章的 URL，系统会通过反查识别其来源公众号并完成订阅。

## Feed 相关

### Mp2RSS 支持哪些 Feed 格式？

支持 **RSS 2.0 / Atom 1.0 / JSON Feed 1.1 / OPML 2.0**。你可以按需选择，不同格式之间内容保持一致。

### 我能在阅读器里订阅"全部公众号合集"吗？

可以。所有格式都提供 `/all` 变体：
- `/feed/<token>/rss/all`
- `/feed/<token>/atom/all`
- `/feed/<token>/json/all`

粘贴到阅读器即可看到所有订阅公众号的统一时间线。

### OPML 怎么下载？

访问 `/feed/<token>/opml` 默认在浏览器中直接渲染显示；在链接末尾加 `?download=1` 参数即可触发浏览器下载为 `.opml` 文件，便于导入其他 RSS 服务。

### Feed 链接多久刷新一次？

Feed 接口默认带 `Cache-Control: max-age=300`（5 分钟），主流阅读器会按该 TTL 自行调度。同时系统为每个 Feed 计算 `ETag` 与 `Last-Modified`，文章未变时可命中缓存。

### 为什么阅读器里突然有一条"会员已到期"的条目？

账户到期后（含试用到期）Feed 里会注入一条显眼的"会员已到期。

## Feed 密钥

### Feed 密钥泄漏了怎么办？

立即登录控制台 → "账户设置" → **重置 Feed 密钥**。重置后旧链接**即时失效**，所有在外部阅读器里配置的旧链接都会失败，请用新链接重新订阅。

### Feed 密钥和 API 的 Bearer Token 是什么关系？

是同一个。Mp2RSS 的 API 直接复用 Feed 密钥作为 `Authorization: Bearer` 凭据。用户在控制台重置 Feed 密钥的同时，也意味着旧的 API 调用凭据同步失效。

## 付费与计费

### 试用期和付费期有什么区别？

没有任何权益差别。

### 怎么看我的剩余试用 / 会员时长？

登录控制台 → "账户设置"，会显示试用剩余天数 / 会员有效期以及近期付费记录。

### 我转账完成后为什么会员没立即开通？

充值采用人工核对方式，需要少量处理时间。

### 我在有效期内续费会浪费剩下的时间吗？

不会。有效期内续费，新时长会**叠加**在当前到期时间之上；只有过期后续费才从续费当日起算。
